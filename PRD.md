# 项目背景
## 市场&竞品分析

## 客户痛点
场景：在浏览器环境中，使用文章、视频等形式学习时。
行为：打开其他笔记软件，同时浏览2个或以上窗口，进行笔记或摘录等操作。
痛点：
- 多窗口视线切换，打断沉浸式学习过程。===悬浮式笔记窗口
- 笔记无法快速溯源。===支持直接获取并保存当前网页链接和标题、作者等信息。

## 上线目标
【1】实现笔记从网页输入到导出的全流程。
【2】用户留存率达到30%以上（初次使用后7天内再次使用视为留存用户）。
【3】Chrome插件市场星级在4.5星以上。

## 产品定位
MVP：轻量级网页笔记工具

# 产品方案
## 用户旅程地图
- 用户跳转到对应网页
- 用户单击浏览器插件栏中的sidenote图标，打开side note窗口
- 用户边看视频/文章边记录
- 用户完成笔记，导出所有笔记内容
- 用户粘贴至其他笔记软件中

## 实体属性

### 睡眠状态（默认）
触发条件：
【1】默认状态
【2】用户完成数据输入和复制后，单击插件窗口右上角的x按钮；重回睡眠状态。
不展示悬浮球，只显示在浏览器插件栏。

### 唤醒状态
触发条件：用户单击浏览器插件栏中的sidenote图标
位置：文本编辑器悬浮在浏览器窗口右侧偏下位置。
大小：宽度为浏览器窗口宽度25%；高度为窗口高度75%，居中。支持用户手动拖拽边框调整大小，位置不可调整。

## 功能详述
### 获取
页面支持获取如下元素则进行捕捉，显示在输入范围顶端，作为补充信息；支持用户手动编辑。
- 获取网页标题&链接：用户在浏览器窗口打开side note时，获取当前浏览器窗口的网页链接和标题；优先级为链接（必需）>=创建时间(必需)>标题
- 获取note创建时间：用户在浏览器窗口打开side note时，获取当前note的创建时间。
示例：
--
url：https://gemini.google.com/share/b3da46a9428a
title：how to use sidenote
created_at：2025-01-23 14:16:45
--
笔记内容

### 记录
- 编辑器：支持实时渲染markdown语法。
- 文本格式：Markdown，以markdown语法为持久性语言，需要支持markdown语法中的标题、加粗、斜体、列表、引用。
- 输入范围固定，当用户输入超过范围时，自动滚动输入页面。

### 存储
本地存储：所有数据暂存至本地，用户执行以下行为后，该标签页插件内容数据将立刻被清空。
【1】关闭插件窗口所在标签页
【2】刷新插件窗口所在标签页
【3】关闭整个浏览器

### markdown语法&渲染
依赖于CodeMirror 6解析
需渲染的markdown语法如下：
---块级语法---
标题：#/##/###… + 空格 → 标题样式
列表：- 、* 、+ 、1. + 空格 → 列表项样式（允许嵌套）
引用：> + 空格 → 引用样式
---行内语法---
加粗：**text** 或 __text__（成对闭合才渲染）
斜体：*text* 或 _text_（成对闭合才渲染）
行内代码：`code`（成对闭合）
链接：[text](url)、自动链接/裸 URL

渲染方式：单层渲染
光标所在行显示原始markdown内容（列表除外），如一级标题显示为# 一级标题内容。

### 导出
一键复制：单击插件窗口右上角的【复制】按钮，将sidenote内所有文本（包含头部信息与正文内容）全部复制完成；弹出提示“复制完成啦”

### light/dark模式切换
一键切换：单击插件窗口右上角的【切换】按钮，切换sidenote的明亮或者夜间模式；默认跟随浏览器模式。

## UI
### 窗口
一个悬浮在玻璃拟态背景上的现代卡片式笔记界面，设计优先考虑排版和易读性。
视觉语言:
表面: 磨砂质感，微圆角 (12-16px)。
排版: Inter 或 San Francisco 字体栈；标题高对比度，元数据中等对比度。
点缀: UI 框架使用柔和色调，用户内容高亮使用鲜艳色。

### 明亮模式
背景为书写纸风格，编辑器右下角要有纸张泛黄打卷或有人写过的潦草笔记，但不能占用到编辑器输入范围，避免吸引用户注意力
浅色模式: 暖调米白 (#F9F9F9) 基底搭配深石板灰文字 (#333333)。

### 深色模式
弹窗其余部分更偏深色rgb(8, 7, 7)，编辑器（用户输入编辑范围）背景色为深炭灰 (#1E1E1E) 基底搭配浅灰文字 (#E0E0E0)。

### 弹窗提示
悬浮的、半透明的胶囊通知，带有背景模糊效果 (backdrop-filter)，并距底部边缘 24px 居中。
动效：淡入淡出。
成功提示：提示背景为淡绿色。
失败提示：提示背景为淡红色。

## 异常边界
### 获取失败
【1】获取网页标题&链接失败：显示为“未获取到哦，可手动输入”。
【2】获取note创建时间失败：显示为“未获取到哦，可手动输入”。
获取失败后支持用户手动编辑未获取/获取错误的url、标题与创建时间。

### 复制失败
复制/导出失败：弹出提示“复制失败咯，请重试”。

### 多笔记场景
用户在多个标签页同时打开多个side note，视为多个独立的note。
